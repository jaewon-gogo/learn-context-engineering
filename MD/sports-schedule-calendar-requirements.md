# 스포츠팀 경기일정 캘린더 웹사이트 프로젝트 요구사항

## 프로젝트 배경 및 문제 인식

10살 아들이 좋아하는 스포츠팀의 경기일정을 확인하기 위해 매번 여러 웹사이트를 방문해야 하는 번거로움이 있습니다. 또한 경기일정이 텍스트 형태로만 제공되어 한눈에 파악하기 어렵습니다. 이를 해결하기 위해 경기일정을 자동으로 수집하고 캘린더 형태로 시각화하여 제공하는 웹사이트가 필요합니다.

## 시스템의 목적 및 개요

이 시스템은 다음과 같은 기능을 제공합니다:

- Google 계정으로 로그인 (사용자 인증)
- 사용자로부터 스포츠 종목과 팀 정보를 입력받음
- 웹에서 해당 팀의 경기일정을 자동으로 크롤링
- 수집한 경기일정을 캘린더 형태로 시각화하여 표시
- 경기일정을 Google 캘린더에 자동으로 전송
- 10살 아이도 쉽게 이해하고 사용할 수 있는 직관적인 UI/UX 제공

  - 입력된 스포츠 종목과 팀에 맞는 웹사이트 URL 매핑

### Google 캘린더 연동 관련 추가 요구사항

#### Google 계정 인증

- **인증 방식**: Google OAuth 2.0
- **장점**: 
  - Apple Developer Program 가입 불필요 (무료)
  - Google Cloud Platform 무료 티어 활용 가능
  - 더 넓은 플랫폼 지원 (Android, iOS, Web)
- **필요한 설정**:
  - Google Cloud Platform 프로젝트 생성
  - OAuth 2.0 클라이언트 ID 및 Secret 발급
  - 승인된 리디렉션 URI 설정

#### Google Calendar API 연동

- **API 사용**: Google Calendar API (REST API)
- **장점**:
  - 잘 문서화된 API
  - 다양한 프로그래밍 언어 지원
  - 풍부한 기능 (알림, 공유, 색상 등)
- **주요 기능**:
  - 이벤트 생성/수정/삭제
  - 캘린더 목록 조회
  - 캘린더 생성
  - 알림 설정 (이메일, 푸시)
  - 캘린더 공유 설정

#### Google 캘린더 특화 기능

1. **캘린더 선택 및 관리**
   - 기본 캘린더 사용 옵션
   - 새 캘린더 생성 옵션 (예: "경기일정" 캘린더)
   - 여러 캘린더 지원 (선택)

2. **알림 및 리마인더**
   - 이메일 알림 (Gmail 통합)
   - 푸시 알림 (모바일 앱)
   - 알림 시점 설정 (경기 전 1일, 1시간, 30분 등)
   - 사용자별 알림 설정

3. **캘린더 공유**
   - 가족/친구와 캘린더 공유
   - 공유 권한 설정 (읽기 전용/편집 가능)
   - 공유 링크 생성 (선택)

4. **이벤트 상세 정보**
   - 경기장 위치 정보 (Google Maps 연동 가능)
   - 상대팀 정보
   - 경기 결과 업데이트 (경기 종료 후)
   - 색상 구분 (홈/원정, 승/패 등)

5. **다중 캘린더 지원** (선택)
   - 팀별로 다른 캘린더 사용
   - 스포츠 종목별 캘린더 분리

#### Google Calendar API 제약사항 및 고려사항

1. **API 할당량**
   - 무료 티어: 일일 1,000,000 쿼리
   - 사용자당 쿼리 제한: 초당 100 쿼리
   - 대량 이벤트 생성 시 배치 처리 필요

2. **비용**
   - Google Calendar API: 무료 (적절한 사용 시)
   - Google Cloud Platform: 무료 티어 제공
   - 과도한 사용 시 비용 발생 가능 (매우 드묾)

3. **보안**
   - OAuth 토큰 안전한 저장 (암호화)
   - API 키는 서버 측에서만 사용
   - 사용자 동의 범위(Scope) 최소화

4. **데이터 동기화**
   - 양방향 동기화 (선택): Google 캘린더에서 수정된 내용 반영
   - 충돌 해결 전략 필요 (선택)

#### Google 캘린더 연동 구현 시 추가 고려사항

1. **OAuth 2.0 Scope 설정**
   - `https://www.googleapis.com/auth/calendar` (읽기/쓰기)
   - `https://www.googleapis.com/auth/calendar.events` (이벤트만)
   - 최소 권한 원칙 적용

2. **API 버전 관리**
   - Google Calendar API v3 사용
   - 향후 버전 업데이트 대응

3. **에러 처리**
   - API 할당량 초과 시 대기 및 재시도
   - 네트워크 오류 시 재시도 로직
   - 사용자 권한 오류 시 재인증 안내

4. **성능 최적화**
   - 배치 요청으로 여러 이벤트 동시 처리
   - 캐싱으로 불필요한 API 호출 최소화
   - 증분 동기화 (변경된 이벤트만 업데이트)

5. **사용자 경험**
   - Google 로그인 버튼 (Google Design 가이드 준수)
   - 캘린더 선택 UI (드롭다운 또는 리스트)
   - 알림 설정 UI (간단하고 직관적)
   - 동기화 상태 표시
