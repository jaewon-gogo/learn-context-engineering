## 시스템 분석 결과 요약

### 백엔드 기능 수
- **총 API 엔드포인트**: 14개
- **서비스 클래스**: 3개 (AuthService, CrawlerService, CalendarSyncService)
- **데이터 모델**: 8개 테이블
- **외부 연동**: 2개 (Google OAuth, Google Calendar API)

### 프론트엔드 기능 수
- **페이지**: 4개 (로그인, 대시보드, 설정)
- **컴포넌트**: 8개 (인증, 선택, 캘린더, 모달 등)
- **커스텀 훅**: 3개 (useAuth, useSchedule, useCalendar)
- **상태 관리**: React Hooks (useState, useEffect, useReducer)

### 중요 기능 수
1. **인증 시스템** (Google OAuth 2.0)
2. **경기일정 조회** (캘린더 UI)
3. **크롤링 시스템** (자동 데이터 수집)
4. **Google 캘린더 동기화**

### 외부 연동 수
- Google OAuth 2.0 서버
- Google Calendar API
- 크롤링 대상 웹사이트 (동적)

## 레벨별 구현 계획

### Level 1: 유닛/모킹 기반 구현 (70%)

#### Backend Level 1
**목적**: 비즈니스 로직 단위의 유닛 구현
**예상 소요 시간**: 5-7일
**구현 건수**: 약 45건

**핵심 대상 기능**:

1. **인증 로직**
   - JWT 토큰 생성/검증 함수
   - 토큰 암호화/복호화 함수
   - OAuth URL 생성 로직
   - 사용자 정보 검증 로직

2. **데이터 처리 로직**
   - 경기일정 데이터 파싱 로직
   - 날짜/시간 형식 변환
   - 데이터 검증 로직
   - 중복 체크 로직

3. **크롤링 로직**
   - HTML 파싱 함수
   - 데이터 추출 로직
   - 에러 처리 로직

4. **캘린더 연동 로직**
   - 이벤트 생성 데이터 변환
   - 이벤트 업데이트 로직
   - Rate Limiting 처리 로직

**필요 환경**: 개발 환경만 필요 (Mock 기반)
**리스크**: 낮음

#### Frontend Level 1
**목적**: 컴포넌트 단위 유닛 구현
**예상 소요 시간**: 5-7일
**구현 건수**: 약 36건

**핵심 대상 기능**:

1. **UI 컴포넌트**
   - 공통 컴포넌트 (Button, Input, Card, Select, Dialog)
   - 로그인 버튼 컴포넌트
   - 스포츠/팀 선택 컴포넌트
   - 캘린더 컴포넌트
   - 경기 상세 모달 컴포넌트

2. **커스텀 훅**
   - useAuth 훅 (Mock API)
   - useSchedule 훅 (Mock API)
   - useCalendar 훅 (Mock API)

3. **이벤트 처리**
   - 폼 제출 처리
   - 입력값 검증
   - 클릭 이벤트 처리

4. **상태 관리**
   - 로컬 상태 관리 (useState)
   - 사이드 이펙트 처리 (useEffect)

**필요 환경**: 개발 환경만 필요 (Mock API)
**리스크**: 낮음

### Level 2: 통합 구현 (20%)

#### Backend Level 2
**목적**: 기능 간 통합 및 연계 구현
**예상 소요 시간**: 7-10일
**구현 건수**: 약 30건

**핵심 대상 기능**:

1. **API + DB 연동**
   - 인증 API + DB 연동
   - 경기일정 조회 API + DB 연동
   - 캘린더 설정 API + DB 연동
   - 트랜잭션 처리

2. **서비스 간 연동**
   - AuthService + CalendarSyncService 연동
   - CrawlerService + ScheduleService 연동
   - 에러 핸들링 체인

3. **외부 API 연동 (개발 환경)**
   - Google OAuth 연동 (개발용)
   - Google Calendar API 연동 (개발용)
   - 크롤링 테스트 (개발용)

**필요 환경**: 개발 DB, 개발용 외부 서비스
**리스크**: 중간

#### Frontend Level 2
**목적**: 다중 컴포넌트 및 API 통합 구현
**예상 소요 시간**: 5-7일
**구현 건수**: 약 24건

**핵심 대상 기능**:

1. **컴포넌트 간 상호작용**
   - 로그인 → 대시보드 플로우
   - 스포츠 선택 → 팀 선택 → 캘린더 표시
   - 경기 클릭 → 상세 모달 표시

2. **API 연동**
   - 실제 API 호출 처리
   - 에러 핸들링
   - 로딩 상태 관리
   - 데이터 캐싱

3. **상태 관리**
   - 전역 상태 관리 (Context API 또는 Zustand)
   - API 응답 상태 동기화
   - 로컬 스토리지 연동

**필요 환경**: 개발 백엔드 서버
**리스크**: 중간

### Level 3: E2E 구현 (10%)

#### Backend Level 3
**목적**: 전체 플로우 기반 백엔드 기능 구현
**예상 소요 시간**: 5-7일
**구현 건수**: 약 20건

**핵심 대상 기능**:

1. **전체 플로우**
   - 사용자 가입 → 로그인 → 캘린더 동기화 전체 플로우
   - 크롤링 → 데이터 저장 → 캘린더 동기화 플로우
   - 에러 복구 플로우

2. **성능 최적화**
   - 쿼리 최적화
   - 캐싱 전략 적용
   - 배치 처리 최적화

3. **보안 강화**
   - 인증/인가 플로우 완성
   - 데이터 보호 로직
   - 입력값 정제 처리

**필요 환경**: 스테이징 환경
**리스크**: 높음

#### Frontend Level 3
**목적**: 사용자 플로우 전체 구현
**예상 소요 시간**: 5-7일
**구현 건수**: 약 15건

**핵심 대상 기능**:

1. **전체 사용자 여정**
   - 로그인 → 팀 선택 → 경기일정 조회 → 캘린더 동기화 플로우
   - 설정 변경 플로우
   - 에러 처리 플로우

2. **크로스 브라우저 대응**
   - Chrome, Firefox, Safari 최적화
   - 반응형 디자인 완성

3. **접근성**
   - ARIA 속성 적용
   - 키보드 네비게이션
   - 10살 아동 친화적 UI

**필요 환경**: 스테이징 환경
**리스크**: 높음
